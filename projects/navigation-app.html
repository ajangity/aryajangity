Navigation App for the Visually Impaired

Indoor assistive navigation system — graph-based routing, computer vision localization, and multimodal accessible interfaces

Active — Feb 2026 Computer Vision Graph Routing Python Assistive Tech

Mission & Motivation

Navigating indoor environments is something most people do without a second thought. For individuals with visual impairments, however, navigating an unfamiliar building — a hospital, a university, a transit hub — can be disorienting, stressful, and unsafe without assistance.

This project aims to change that. We are building an indoor assistive navigation system that enables visually impaired users to receive room-level guidance through structured routing and accessible, multimodal interfaces — voice, haptic, and audio cues that work without requiring sight.

This is not just a software problem. It requires rethinking how spaces are represented, how users interact with technology under uncertainty, and how computer vision can be deployed reliably in real-world building environments.

How It Works

The system operates in two phases: environment mapping and real-time navigation.

Graph-Based Indoor Routing

We represent building layouts as node-edge networks — rooms, corridors, stairs, and landmarks are modeled as nodes, with edges weighted by traversal difficulty, distance, and accessibility constraints. Shortest-path algorithms compute optimal routes based on the user’s current position and destination.

This approach is inherently flexible: it can handle multi-floor buildings, one-way corridors, and dynamically blocked paths without requiring a complete re-map.

Computer Vision Localization

Rather than relying on GPS (which is unreliable indoors), the system uses computer vision to determine the user’s location within the building. Visual cues — doorframes, signage, floor patterns, and landmarks — are processed to estimate position and orientation, which is then matched to the map graph to confirm the user’s node.

Multimodal Accessible Interface

Navigation instructions are delivered through accessible output modalities: spoken audio directions, earcon audio cues for turns and landmarks, and (in future iterations) haptic feedback for key decision points. No visual display is required.

Research Collaboration

This project is developed in collaboration with Professor Roberto Manduchi and the UCSC Computer Vision Lab. Prof. Manduchi is a leading researcher in assistive technology and computer vision, with decades of work in navigation systems for the visually impaired.

My contributions span system architecture design, graph routing implementation, and UI development for real-world deployment. Working within this research environment provides direct access to expertise in accessibility and vision-based sensing that shapes the technical direction of the project.

Key Technical Challenges

  • Reliable indoor localization without GPS in dynamic, visually variable environments
  • Building compact, accurate map representations for large multi-floor spaces
  • Designing interfaces that are fully accessible and usable without sight
  • Handling partial occlusions and lighting variation in computer vision pipeline
  • Ensuring the system degrades gracefully when localization confidence is low
  • Minimizing computational latency for real-time navigation feedback

Current Status

The project is actively under development as of early 2026. Current focus areas include refining the graph routing engine, validating CV localization in UCSC campus buildings, and designing the accessible user interface for initial user testing.

Future Directions

  • Integration of LiDAR or depth sensing for robust 3D localization
  • Machine learning–based landmark detection for improved CV accuracy
  • Mobile app interface for deployment on standard smartphones
  • User testing with visually impaired participants and accessibility researchers
  • Expansion to multi-floor navigation with elevator and staircase routing
  • Dynamic obstacle detection and real-time path re-routing
